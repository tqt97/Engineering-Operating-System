
# Hệ thống quy tắc vận hành kỹ thuật của team

Đây không phải:

- **Tài liệu học lập trình.**
- **Wiki kiến thức.**
- **Nơi giải thích lý thuyết.**

Đây là **quy tắc để team làm việc với nhau theo 1 chuẩn chung và hiệu quả hơn**.

---

## I. Mục tiêu

Tài liệu được tạo ra để:

- Chuẩn hóa cách team ra quyết định kỹ thuật
- Giảm sai lệch giữa các developer
- Giảm bug production
- Giảm phụ thuộc vào cá nhân quản lý, các anh/em có nhiều kinh nghiệm hơn
- Tăng tốc hòa nhập cũng như phát triển dự án
- Tạo một cách làm việc thống nhất

Toàn bộ tài liệu được chia theo 4 lớp:

```txt
Think → Define → Build → Run
```

| Layer | Mục đích |
|------|--------|
| Think | Tư duy kỹ sư |
| Define | Làm rõ requirement |
| Build | Quy trình phát triển |
| Run | Vận hành production |
